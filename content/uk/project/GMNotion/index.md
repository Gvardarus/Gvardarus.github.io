## 1. Головні проблеми

У даному кейсі розглядається випадок коли користувач приймає на електронну пошту повідомлення від клієнтів. Це може бути онлайн-магазин що приймає заявки на купівлю товару, або це просто пошта на яку приходить велика кількість повідомлень.  Основна проблема полягає у "ручному" копіюванні даних та перенесення їх у інше середовище для опрацювання що є дуже монотонним, рутинним та часовитратним процесом.

## 2. Задачі

Створити та налаштувати систему таким чином щоби позбавити користувача рутинної роботи взагалі, автоматизувавши всі дії з перенесенням даних.  
Для реалізації було вибрано платформу Make(Integromat) через наявність гнучких у налаштуванні модулів-шаблонів

## 3. Поетапне рішення (скріншоти)

1. Створення нового проэкту у Make. Вибрав модуль Gmail (Watch emails) так як цей модуль підходить більш за все у даній ситуації. Він запускає виконання сценарію як тільки на пошту приходить нове повідомлення.

![вибір модуля Gmail](imgproj/modul.jpg)

2. Створив нову таблицю у робочому середовищі Notion. Додав колонки для необхідного типу інформації та фільтрацію по типу даних (дуже зручно сортувати список по окремому типу). Також можна налаштувати можливість перегляду та редагування таблиці кількома учасниками команди у реальному часі.

![таблиця у Notion](imgproj/tablenotion.jpg)

3. У налаштуваннях модуля Gmail вибрав електронну пошту на яку будуть приходити повідомлення. Папка для перегляду - Inbox (вхідні). Саме звідти повідомлення будуть відправлятися до середовища Notion.

![налаштування модуля Гмейл](imgproj/gmailset.jpg)

4. Вибираємо модуль Notion (Create a Database item Legacy) щоби при отриманні нового повідомлення створювати кожен раз новий рядок у базі даних Notion. В налаштуваннях модуля вибираємо через ID пошук потрібну базу даних та вказуємо тегами який саме тип даних з повідомлення ми очікуємо бачити у своїй базі даних. У відповідну колонку вказуємо відповідний тег.

![налаштування модуля Гмейл](imgproj/notionset.jpg)

## 4. Результати

Після налаштування модулів можемо перевірити результат системи

Як тільки у "вхідних" електронної скриньки з'являється нове повідомлення, тригер-модуль Gmail запускає сценарій. Нове повідомлення та ті типи даних які ми вказали у налаштуваннях модуля Notion будуть автоматично заповнювати таблицю

![результат](imgproj/all.gif)

Підсумок виконаної роботи:

Було спроєктовано та реалізовано no-code автоматизацію, яка дозволяє автоматично переносити вхідні повідомлення з Gmail до бази даних Notion. Рішення усуває ручне копіювання даних, зменшує ризик помилок та забезпечує зручний доступ до структурованої інформації в одному середовищі.

Інструменти:
- Gmail API documentation
- Notion API documentation
- Make(Integromat)
- Notion workspace
